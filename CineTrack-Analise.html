Documento técnico gerado a partir do código-fonte do repositório em 28/01/2026.
Este material descreve o objetivo do sistema, cada módulo, fluxos principais e o passo a passo para colocar tudo em operação.

Objetivo do sistema

O CineTrack é um catálogo pessoal de filmes e séries que centraliza:

  • registro e consulta de mídias (filmes/séries)
  • acompanhamento de progresso por temporada/episódio
  • avaliações e comentários do usuário
  • estatísticas de consumo
  • sincronização via scraping e associação assistida

Em resumo: ele funciona como um hub pessoal de descoberta, acompanhamento e histórico de conteúdo audiovisual.

Diferenciais atuais (confirmados no código)

  • Scraping automatizado de lançamentos via SceneSource, com logs e status de sincronização
  • Ratings integrados do IMDb e Rotten Tomatoes (Tomatometer/Popcornmeter) no catálogo
  • Acesso remoto seguro via Azure Relay sem expor portas no servidor local
  • App híbrido moderno com .NET MAUI + Blazor (UI responsiva e rápida)

Visão geral da arquitetura

[MOBILE (MAUI + Blazor)]
        │  HTTPS (Azure Relay + SAS)
        ▼
[AZURE RELAY] ──► [RelayListener (Windows Service)] ──► [CineTrack.API] ──► [SQL Server]
                                                            ▲
                                                            │
                                                   [Shared (Models/Repos)]

[Azure Functions] ──► [Scrapers (SceneSource/IMDb)] ──► [SQL Server]
  • CineTrack.Mobile: aplicativo MAUI/Blazor com UI e navegação local
  • CineTrack.API: API REST (ASP.NET Core) com endpoints de mídia, usuário, sync e estatísticas
  • CineTrack.Shared: modelos, repositórios Dapper e serviços de scraping compartilhados
  • CineTrack.Functions: Azure Functions para scraping automático e status
  • CineTrackRelayListener: ponte Azure Relay → API local
  • CineTrackTokenGen: gerador de SAS token do Azure Relay
  • SQL Server: banco com tabelas, views, SPs e dados de exemplo

Por que Azure Relay?

O Azure Relay permite acesso remoto ao servidor sem abrir portas no firewall.

O RelayListener abre uma conexão de saída (outbound) para a nuvem, e o app se conecta ao Relay usando um token SAS.

Resultado: o SQL Server e a API podem ficar somente na rede local, com acesso externo controlado.

Detalhamento por módulo

CineTrack.API

Papel: camada REST para consumo pelo app e para operação de sync.

Principais arquivos: CineTrack.API/Program.cs, CineTrack.API/Controllers/*.cs

Responsabilidades:

  • servir catálogo e detalhes de mídia
  • gerenciar status, avaliação e progresso do usuário
  • expor estatísticas e logs
  • disparar scraping manual

Endpoints principais:

  • GET /api/midias (filtros por status e tipo)
  • GET /api/midias/search?termo=
  • GET /api/midias/{id}
  • GET /api/midias/{id}/temporadas / episodios
  • PUT /api/usuario/midia/{id}/status
  • PUT /api/usuario/midia/{id}/nota
  • GET /api/estatisticas
  • POST /api/sync/scraping/executar
  • GET /health

Observações técnicas:

  • CORS liberado para qualquer origem
  • Swagger habilitado em dev e produção
  • Kestrel configurado em http://localhost:5050 via appsettings.json

CineTrack.Mobile

Papel: app MAUI com UI em Blazor (WebView).

Principais arquivos: MainPage.xaml, Components/Pages/*.razor, Services/CineTrackApiClient.cs

Fluxo de UI: Home, Search, Detail, Stats, Sync.

Integração: usa Azure Relay via BaseUrl e SasToken no header ServiceBusAuthorization.

Observações: páginas de template (Counter/Weather) não são usadas no fluxo atual.

CineTrack.Shared

Papel: biblioteca comum com modelos, enums, repositórios e scrapers.

Repositórios (Dapper): Midia/Temporada/Episodio/Usuario/Sync.

Scrapers: SceneSource e IMDb.

Ponto de atenção: dependem do HTML externo (podem quebrar).

CineTrack.Functions

Papel: automação de scraping e status.

  • Timer a cada 30 min: 0 */30 * * * *
  • POST /api/scraping/trigger
  • GET /api/scraping/status
  • GET /api/health

CineTrackRelayListener

Papel: serviço Windows que recebe requisições do Azure Relay e encaminha para a API local.

Configuração atual (hardcoded): RelayConnectionString, BaseUrl (http://localhost:5050), HybridConnectionName.

CineTrackTokenGen

Papel: gerar SAS token para o Azure Relay (expiração de 50 anos).

Banco de dados (CineTrackDb)

Script: CineTrack-Database.sql

Tabelas principais:

  • Midia
  • Temporada
  • Episodio
  • UsuarioMidia, UsuarioEpisodio
  • AssociacaoSceneSource
  • LogCaptura
  • Configuracao

Views: ViewMidiaCompleta, ViewTemporadaCompleta, ViewEpisodioCompleto, ViewAssociacoesPendentes, ViewEstatisticas.

SPs: sp_ObterSugestoes, sp_MarcarEpisodioAssistido, sp_ConfirmarAssociacao, sp_RejeitarAssociacao.

Fluxos principais

Catálogo e navegação

  1. App chama GET /api/midias ou search
  2. API consulta ViewMidiaCompleta
  3. UI renderiza cards na Home/Search

Marcar episódio/temporada

  1. App chama POST /api/usuario/episodio/{id}/assistido
  2. sp_MarcarEpisodioAssistido atualiza episódio e status da mídia
  3. UI atualiza progresso

Scraping e associação

  1. Azure Function ou API dispara scraping
  2. SceneSourceScraper captura itens recentes
  3. Insere em AssociacaoSceneSource como pendente
  4. Usuário confirma/rejeita no app (Sync)

Configurações e segredos

  • CineTrack.API/appsettings.json
  • CineTrack.Functions/local.settings.json
  • CineTrackRelayListener/RelayListenerService.cs (hardcoded)
  • CineTrack.Mobile/Services/CineTrackApiClient.cs (hardcoded)

Valores atuais: SQL Server CTRAN01, DB CineTrackDb, API http://localhost:5050, Azure Relay cinetrack-relay.servicebus.windows.net.

Recomendação: mover tokens e strings para variáveis de ambiente/Secret Manager.

Passo a passo para rodar localmente

1) Pré-requisitos

  • .NET 10 (preview)
  • SQL Server (local ou remoto)
  • Azure Functions Core Tools (opcional)
  • .NET MAUI workload

2) Criar banco

  1. Criar a base CineTrackDb no SQL Server
  2. Executar CineTrack-Database.sql

3) Configurar connection strings

  • CineTrack.API/appsettings.json → DefaultConnection
  • CineTrack.Functions/local.settings.json → SqlConnectionString

4) Rodar API

dotnet run --project CineTrack.API

API sobe em http://localhost:5050 e Swagger em /swagger.

5) Rodar Functions (opcional)

dotnet run --project CineTrack.Functions

6) Rodar Relay Listener (opcional)

dotnet run --project CineTrackRelayListener

7) Gerar/atualizar SAS Token do Relay

dotnet run --project CineTrackTokenGen

Copiar token para CineTrack.Mobile/Services/CineTrackApiClient.cs.

8) Rodar Mobile

dotnet build CineTrack.Mobile

Roteiro de implantação (produção/servidor Windows)

Estrutura sugerida para ambiente de servidor (sem Azure Functions).

1) Publicar API

dotnet publish CineTrack.API -c Release -o C:\CineTrack\API

2) Publicar RelayListener (Windows Service)

dotnet publish CineTrackRelayListener -c Release -r win-x64 --self-contained -o C:\CineTrack\RelayListener

3) Instalar serviço do RelayListener

Abra um terminal como Administrador e execute:

cd C:\CineTrack\RelayListener
.\install-service.bat

4) Gerar token SAS e atualizar o app

dotnet run --project CineTrackTokenGen

Cole o token em CineTrack.Mobile/Services/CineTrackApiClient.cs.

Configuração do Azure Relay (passo a passo)

  1. Acesse o portal do Azure: https://portal.azure.com
  2. Crie um Relay Namespace (ex.: cinetrack-relay)
  3. Crie uma Hybrid Connection chamada cinetrack-bridge
  4. Em Shared access policies, crie a policy ListenSend com permissões Listen + Send
  5. Copie a Connection string – Primary key e configure no RelayListenerService.cs
  6. Verifique em Hybrid Connections > Listeners se aparece 1 conectado

Operação diária

  • Health check: GET /health
  • Scraping manual: POST /api/sync/scraping/executar
  • Confirmar associações no menu Sync do app
  • Consultar logs em /api/sync/logs

Pontos de atenção

  • Segurança: chaves do Azure Relay estão hardcoded no código
  • Scraping: depende de HTML externo (IMDb/SceneSource podem mudar)
  • Portas: API configurada em 5050, mas launchSettings usa 59267/59268
  • Preview: .NET 10 exige tooling atualizado

Troubleshooting (baseado no PDF legado)

  • TokenMissingOrInvalid: o token SAS foi gerado com URL incorreta ou expirou → gere novamente com https://cinetrack-relay.servicebus.windows.net/cinetrack-bridge
  • 404 via Relay: o RelayListener não está rodando ou o HybridConnectionName difere do app → verifique o serviço com Get-Service CineTrackRelayListener

Checklist rápido (operacional)

  • [ ] Banco criado e script aplicado
  • [ ] API respondendo em /health
  • [ ] App conectado (Sync sem erro)
  • [ ] Relay Listener ativo (se uso remoto)
  • [ ] Scraping funcionando (logs aparecem)

Glossário rápido

  • AssociacaoSceneSource: item capturado aguardando validação

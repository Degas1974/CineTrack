02 · Placeholders

Onde ficam e o que trocam

Todos os dados fictícios do protótipo estão em src/data.jsx. Edite esse único arquivo para testar cenários reais sem tocar nas telas.

src/data.jsx → MEDIA[]
Array de filmes e séries com 12 itens de exemplo
PLACEHOLDER
poster / backdrop
URLs de imagem do TMDB — formato https://image.tmdb.org/t/p/w500/{path}. Substitua pelo campo da sua API (ImagemUrl ou ImagemExibicao).
poster: POSTER('/lFf6LLrQjYldcZItzOkGmMMigP7.jpg') // ↓ no seu código: poster: midia.ImagemUrl ?? midia.ImagemExibicao
status
Valores válidos: 'Assistindo', 'Assistido', 'Pendente', 'Abandonado'. Devem bater com os StatusMidia do seu enum.
imdb / tomato / popcorn
Ratings públicos — imdb é decimal (ex: 8.7), tomato/popcorn são inteiros percentuais (ex: 95). Correspondência direta com ImdbRating, Tomatometer, Popcornmeter da sua VM.
progress
Objeto { watched: 14, total: 19 } — só presente em séries em andamento. Mapeia para EpisodiosAssistidos / TotalEpisodios.
user
Nota do usuário em estrelas (1–5). null se não avaliado. Corresponde a EstrelaUsuario.
src/data.jsx → STATS{}
Objeto de estatísticas do usuário — usado em StatsScreen
PLACEHOLDER
horasAssistidas
Total de horas assistidas (inteiro). Exibido como hero no variant B. Vem do seu EstatisticasVM.HorasAssistidas.
generos[]
Array { name, count, pct } — distribuição de gêneros. Precisa ser calculado no backend ou derivado dos gêneros das mídias. Não existe ainda na sua VM.
generos: [ { name: 'Drama', count: 42, pct: 38 }, { name: 'Sci-Fi', count: 28, pct: 25 }, ... ]
atividade[]
12 pontos mensais de atividade { m: 'Mai', v: 8 } para o gráfico de barras. Precisará de endpoint próprio.
topRated[]
Array de IDs das mídias com maior nota do usuário. Resolvido localmente via FIND_MEDIA(id).
src/data.jsx → SYNC_DATA{}
Estado de sincronização — usado em SyncScreen (4 abas)
PLACEHOLDER
pendentes[]
Capturas aguardando confirmação. Cada item tem capturado (título bruto), match (título do banco), confianca (0–100) e releases[]. Mapeia para AssociacaoPendenteVM.
logs[]
Array de logs: { tipo: 'sucesso'|'erro'|'aviso'|'info', data, msg }. Vem de LogCaptura.
diagnostico[]
Resultados do parser: claridade (muito-claro / claro / moderado / ambiguo), score, modo, resumo. Mapeia para DiagnosticoParserVM.
03 · Navegação

Interagindo com o protótipo

1
Abra CineTrack.html no browser
O design canvas carrega automaticamente com todas as telas lado a lado. Use scroll e pan (arrastar o fundo) para navegar entre as seções A · Conservador e B · Ousado.
2
Abra qualquer artboard em fullscreen
Passe o mouse sobre um frame de celular — aparece o ícone ⤢ no canto superior direito. Clique para abrir fullscreen. Use ← → para navegar entre artboards e Esc para fechar.
3
Interaja com os frames
Toque qualquer card de mídia para abrir o Detail. A bottom nav alterna entre Home, Search, Stats e Sync. Na tela Sync, as 4 abas (Status, Pendências, Logs, Diagnóstico) são clicáveis.
4
Use o painel de Tweaks
Clique no botão Tweaks no canto inferior da tela. Alterne a densidade entre Confortável e Compacto — o layout de todos os frames atualiza em tempo real.
5
Edite os placeholders
Abra src/data.jsx e substitua qualquer item do array MEDIA ou do objeto STATS. Recarregue o browser — nenhum build necessário.

05 · Design System

Tokens e paleta

Todos os valores ficam no objeto THEME em src/primitives.jsx. Edite lá para mudar o visual globalmente.

Superfícies
bg · bg1 · bg2 · bg3 — camadas warm-black
Accent
gold · red · success · warning
Fraunces
THEME.serif — títulos, displays, hero numbers. Elegância editorial.
JetBrains Mono
THEME.mono — metadados, ratings, labels técnicos, contadores.
💡 Para mudar a cor de accent do app inteiro (de dourado para, ex., azul), edite apenas THEME.gold em src/primitives.jsx. Todos os componentes herdam o valor.
✓ Os pôsteres vêm de image.tmdb.org/t/p/w500/ — substitua pelo seu CDN trocando a função POSTER() no topo de src/data.jsx.
